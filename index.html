Intelligence is a product decision, not a feature checklist.

Every application follows the same discipline:

  2. 02Keep the core deterministic
  3. 03Use AI for ambiguity or variation
  4. 04Protect the experience from model latency
  5. 05Minimize the cost of keeping it alive

07 · Honest constraints

Low tether is a tradeoff,
not an absence of limits.

Design the failure path before the demo.

CONSTRAINTDESIGN RESPONSE
Free-tier capacity

Projects can pause or compete for limited active slots.

Health manifest + revival runbook

Know what is live, what is sleeping, and how to restore it.

Model latency + variance

Responses are slower and less predictable than local logic.

Async UI + schema + fallback

Keep the workflow responsive and every result correctable.

Managed-service coupling

Convenience depends on platform limits, APIs, and policies.

Document boundaries + exports

Keep data ownership and replacement paths explicit.

Live-demo dependency

Auth, frames, networks, or external services can fail.

Live → recorded → static fallback

Preserve the product story even when infrastructure is unavailable.
